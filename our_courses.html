Our quality assurance course

The materials in this course are based on and support the ISTQB Foundation Syllabus 2018 program, which is the basis for obtaining the Foundation International Certificate in Software Testing. Training takes place on real projects in our company. The resources are carefully tailored to the specific needs of the students, the instructors and the course. A lot of practice and the tremendous experience of our teachers will help you quickly master your dream profession.

The course covers the basic concepts of test design, methodology of test case development, approaches to designing test plans and writing test scripts, as well as features of test plans for different types of testing.

We describe the basic methods of functional and configuration testing based on the functional and configuration requirements of the software product. The course considers major approaches to writing the test design. It covers the concepts of functional and configuration testing as well as methods of testing reliability and stability.

Fundamentals of testing:

  • What is testing?
  • Why is testing necessary?
  • Seven testing principles.
  • Test process.
  • The psychology of testing.

Testing throughout the software development life cycle.

  • Software development life cycle models;
  • Test levels;
  • Test types;
  • Maintenance testing.

Static techniques:

  • Static techniques and test process;
  • Review process;

Test techniques:

  • Categories of test techniques;
  • Black-box test techniques;
  • White-box test techniques;
  • Experience-based test techniques.

Test management:

  • Test organisation;
  • Test planning and estimation;
  • Test monitoring and control;
  • Configuration management;
  • Risk and testing;
  • Defect management;

Tool support for testing:

  • Test tool considerations;
  • Effective use of tools.

Basic technical skills:

  • Web services testing:

    • Basics of web technologies (client-server). Network, transmission protocols Web application structure;
    • IP address, DNS name, DHCP/DNS server;
    • What is REST and SOAP;
    • What is an API and how to view requests and their execution time in the browser?

  • HTML. СSS:

    • What is CSS, why is it needed, and how can it be written;
    • What are attributes, why are they needed, which are the most widely used tag attributes (class);
    • Scripts on an HTML page, why are they needed as they are connected?

  • SQL:

    • Working with SQL Databases;
    • Relational databases, normal forms;
    • Database server concept;
    • Practice writing queries with table management;
    • Logical operators (+ where + like);
    • Aggregation functions (count/sum/max/average);
    • Join practice

  • API testing. Postman:

    • SOAP;
    • REST;
    • Postman as a utility for API testing.

Important: Training takes place on real projects in our company.

Additionally: Master class by folding a resume.

Our project management course

The PM's primary duty and responsibility is to bring the customer's idea to realization within the stipulated timeframe using existing resources. The main task of a business analyst is to identify the customer's business problems and find the most effective solution.

Professionals with skills in both of these professions are invaluable in the labor market. Combining the roles of PM and BA is a practical and reasonable move not only in terms of the project budget or the ratio of time/resources spent on finding one specialist instead of two, but also a very successful concentration of the vision of all the details and the most complete picture of the project in one hand.

On the course you will get: key tools to manage a project of any complexity, experience in requirements gathering and specification, learn effective approaches to estimating tasks and projects.

Training takes place on real projects in our company. A lot of practice and the tremendous experience of our teachers will help you quickly master your dream profession.

Advantages of the profession: high demand in the market, decent pay with clear prospects and an almost endless path of development.

Module 1: Introduction to Project Management

  • Intro. IT Market. Types of companies. Types of IT products (Project, Product, SLA, Startup). Project triangle.
  • Project Management Framework. Project, Product, Program, Portfolio.
  • Project Life Cycle and Phases. Types of SDLC: Predictive, Agile (Scrum, Kanban), Hybrid. How to select methodology for the project.
  • Roles and Responsibilities of a Project Manager. Difference between roles in Agile/Predictive projects.
  • Stakeholder Identification and Analysis. Stakeholder matrix. Work with Confluence.

Testing throughout the software development life cycle.

  • Software development life cycle models;
  • Test levels;
  • Test types;
  • Maintenance testing.

Module 2: Predictive lifecycle. Project Initiation and Planning

  • Defining Project Objectives and Scope. Project Charter.
  • Work Breakdown Structure (WBS). Requirements gathering techniques. Scope creep. Gold plating
  • Setting up project in Jira
  • Risk Identification and Initial Planning
  • Project Kick-off
  • Types of contracts and risks of working with each type

Module 3: Predictive lifecycle. Project Time and Cost Management

  • Creating a Project Schedule using Gantt Charts and Network Diagrams
  • Estimating Activity Durations and Resource Requirements
  • Budgeting and Cost Estimation
  • Cost Control and Earned Value Management

Module 4: Project Quality and Communication Management

  • Defining Project Quality Standards
  • Quality Assurance vs. Quality Control
  • Effective Communication Planning and Execution
  • Conflict Resolution and Stakeholder Management. Team motivation, constructive feedback.

Module 5: Project Execution, Monitoring, and Closure

  • Managing Project Execution and Team Performance
  • Monitoring and Controlling Project Progress
  • Change Management and Scope Control
  • Project Closure and Lessons Learned

Module 6: Agile Project Management

  • Agile Principles and Values: Agile Manifesto and its principles, difference between traditional and agile project management approaches
  • Agile Methodologies and Frameworks: Scrum vs Kanban
  • Scrum roles, ceremonies, and artifacts
  • Kanban principles and practices, metrics
  • Agile Project Initiation: project vision, goals, and objectives, product backlog and prioritizing user stories, acceptance criteria, Definition of Done
  • Estimating and project planning in an agile context, identifying stakeholders and their roles in agile projects
  • Sprint planning and backlog refinement, daily stand-ups, iterative development and continuous integration
  • Monitoring and adapting project progress using agile metrics
  • Sprint reviews/demo and retrospectives, delivering value and ensuring customer satisfaction
  • Soft Skills. Communication, team motivation, constructive feedback. Communication with customer. Conflict resolution
